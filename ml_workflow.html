What is Isolation Forest? (Simple Explanation)
🌳 The Simple Idea — Apple Basket

Imagine you have 1000 apples in a basket. 950 are normal apples. 50 are rotten (anomalies).

If you randomly divide the basket, the rotten apples get separated FIRST because they look different from the crowd. Normal apples need many more cuts to isolate.

Isolation Forest uses this same idea with sensor data — unusual readings (anomalies) get isolated in fewer random splits than normal readings.

✅ Normal Reading

Smoke = 100, Gas = 200, Vib = 1.2

→ Blends in with crowd

→ Needs MANY cuts to isolate

Result: NORMAL (Score = +1)

🚨 Anomaly Reading

Smoke = 2000, Gas = 900, Flame = 1

→ Stands out from crowd

→ Isolated in very FEW cuts

Result: ANOMALY (Score = −1)

🎯 Key Advantage: Isolation Forest does NOT need labeled data. You don't need to tag "this was a fire incident." It learns what NORMAL looks like automatically from sensor history.

Isolation Forest — 7 Detailed Algorithm Steps
  • 1
    COLLECT TRAINING DATA from Database
    When Flask starts, it fetches the last 2,000 sensor rows from MySQL to learn what normal looks like.
    If the database is empty, it waits — the model trains itself once enough live packets arrive.
    SELECT smoke, gas, vib, distance, flame, gateway_rssi, gateway_snr FROM sensor_data ORDER BY id DESC LIMIT 2000
  • 2
    CLEAN & PREPARE FEATURES
    Bad values are fixed before training:
    • Distance = 0 cm → replaced with 999 (sensor error / object too far)
    • RSSI = 0 dBm → replaced with −120 dBm (no signal reading)
    • Then all 7 features are scaled to the same range using StandardScaler
    7 features = [smoke, gas, vib, distance, flame, rssi, snr] StandardScaler → normalises each feature to mean=0, std=1 (so smoke:2000 and vib:9.5 are compared fairly)
  • 3
    BUILD 150 ISOLATION TREES
    The algorithm grows 150 random decision trees.
    Each tree: randomly picks one feature + a random split value → divides data into two halves.
    Repeats until each data point is alone (isolated).
    Fewer splits to isolate = more anomalous.
    IsolationForest(n_estimators=150, contamination=0.08, random_state=42) model.fit(X_scaled) ← learns NORMAL patterns from 2000 rows Saves to: ml_isolation_forest.joblib
  • 4
    SCORE EVERY INCOMING PACKET (Real-Time)
    When a new sensor packet arrives (every 2 seconds):
    Step A: Extract 7 features from the packet
    Step B: Apply same scaler (must match training)
    Step C: Pass through all 150 trees → compute average isolation depth
    Step D: Decision function converts to a score
    score = model.decision_function(X_scaled) ← negative = anomalous label = model.predict(X_scaled) ← -1 = anomaly, +1 = normal confidence = (-score + 0.5) × 100 ← convert to 0–100%
  • 5
    CLASSIFY: NORMAL / WARNING / CRITICAL
    Based on the prediction and confidence level, assign a human-readable label:
    if label == +1 → NORMAL (all good, no alert) if label == -1 AND confidence < 70% → WARNING (monitor closely) if label == -1 AND confidence >= 70% → CRITICAL (immediate action!)
  • 6
    BROADCAST RESULT via SSE to Dashboard
    ML result is pushed instantly to every connected browser tab via Server-Sent Events (SSE).
    Dashboard AI panel updates in under 1 second — no page refresh needed.
    SSE event: { "type": "ml_score", "node_id": "NODE_01", "ml_label": "CRITICAL", "ml_confidence": 87.3, "ml_reason": "Smoke=2100, Flame=YES, Gas=950" }
  • 7
    AUTO-RETRAIN EVERY 200 PACKETS
    After every 200 new packets, the model re-trains itself in the background thread.
    This keeps the model updated with the latest factory conditions.
    Scoring never stops — retraining runs in parallel without any interruption.
    Packet count ≥ 200 → background thread → fetch 2000 rows → retrain New model saved → ml_isolation_forest.joblib updated ml_model.py: maybe_retrain() called after every score_packet()
