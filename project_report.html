01
Executive Summary

The Adaptive LoRa and Edge AI Framework with Machine Learning-Based Predictive Adaptive Communication for Smart Industrial Monitoring is a full-stack Industrial IoT system deployed at ApexChem Blending Works — a small chemical blending plant handling flammable solvents, drum filling operations, and hazardous storage zones.

The system deploys 3 ESP32 sender nodes + 1 LoRa gateway + 1 ESP32-CAM across 3 hazard zones. Each node transmits multi-sensor telemetry (smoke, gas, flame, vibration, proximity, GPS) over LoRa 433 MHz every 2 seconds. The gateway forwards data to a Flask/MySQL backend via WiFi, which serves a React dashboard with sub-1-second Server-Sent Events (SSE) real-time updates. An edge ML layer detects anomalies at the node level, and the adaptive spreading factor algorithm automatically optimises LoRa communication parameters.

Distance Estimation — RSSI & SNR (from Notes)
📝 From Project Notes: Distance is estimated from RSSI and supplemented by SNR. The log-distance path loss model is used. Gateway enriches every packet with gateway_rssi and gateway_snr from the SX1278 receiver.
📶 RSSI Scale — dB Reference Values
RSSI Value (dBm)MeaningSystem Action
−40 dBm🟢 Very Close — Node right next to gatewaySF7 selected, full speed
−55 to −70 dBm🔵 Good Range — Normal operationSF7 / SF9 adaptive
−85 dBm🟡 Minimum Threshold — Marginal linkWarning log + SF upgrade
−100 dBm🟠 Weak — Through walls / obstructionsSF11 selected
−120 dBm🔴 Very Far — Edge of reliable rangeSF12 max range, alert sent
📡 SNR (Signal-to-Noise Ratio) Reference
SNR ValueLink QualityDistance Estimate
> 10 dBExcellentVery reliable (±5%)
5 – 10 dBGoodReliable (±15%)
0 – 5 dBMarginalLess accurate (±30%)
< 0 dBPoor — high packet lossUnreliable — alert
📐 Distance Estimation Formula (Log-Distance Path Loss Model)
d = 10 ((Pt − RSSI) / (10 × n))
d = estimated distance (meters)  |  Pt = TX power = +17 dBm  |  RSSI = received signal strength (dBm, negative)  |  n = path loss exponent
EnvironmentPath Loss Exponent (n)Condition
Free space / Line-of-sightn = 2.0Outdoor, no obstructions
Open indoor factory floorn = 2.5 – 3.0Large open hall, few metal racks
Dense factory / chemical plantn = 3.5Metal structures, machinery, walls
Through multiple wallsn = 4.0 – 4.5Concrete/brick walls, multi-floor
Example Calculation (ApexChem factory, n=3.5):
Pt = +17 dBm, RSSI = −75 dBm → d = 10((17−(−75))/(10×3.5)) = 10(92/35) = 102.63 ≈ 424 m

Adaptive Spreading Factor — LoRa (Notes Image 2, Item ⑨)
📡 17.1 — What is Spreading Factor (SF)?

LoRa's Spreading Factor (SF7–SF12) controls the trade-off between range and data rate. Higher SF = longer range but slower data rate and more air time. The system automatically selects SF based on real-time RSSI and SNR readings from the gateway.

SFData RateAir Time (100B)Range (indoor factory)Use Case
SF7~5.5 kbps56 ms50–100 mDefault — close nodes, ZONE_1/2/3
SF9~1.75 kbps206 ms100–200 mNormal operation with mild obstructions
SF10~980 bps371 ms150–250 mThrough 1–2 concrete walls
SF11~440 bps823 ms200–300 mLong corridor / outdoor campus
SF12~244 bps1,319 ms300 m+Maximum range / emergency fallback
⚙️ 17.2 — Adaptive SF Algorithm
ALGORITHM: Adaptive Spreading Factor Selection

1. Read gateway_rssi and gateway_snr from latest LoRa packet
2. IF RSSI > −70 dBm AND SNR > 10 dB:
    → Set SF = SF7 (maximize speed, close node)
3. ELSE IF RSSI > −85 dBm AND SNR > 7 dB:
    → Set SF = SF9 (balance range/speed)
4. ELSE IF RSSI > −100 dBm AND SNR > 3 dB:
    → Set SF = SF10 (prioritize reliability)
5. ELSE IF RSSI > −115 dBm:
    → Set SF = SF11 (long range mode)
6. ELSE:
    → Set SF = SF12 (maximum range — emergency)
7. Log SF change with timestamp → Dashboard Network Health Panel
8. Update node configuration via LoRa downlink / firmware param
